# Schema do Supabase para o Portal de Links

Este arquivo resume as tabelas usadas pelo backend em [backend/database.py](backend/database.py) e o que precisa existir no Supabase antes de apontar o projeto para produção na Vercel.

## Visao geral

O banco tem 4 tabelas principais:

- `empresas`: cadastro das empresas atendidas.
- `links`: links cadastrados por empresa.
- `chamados`: tickets/solicitacoes por empresa.
- `agenda_eventos`: eventos da agenda interna.

Os relacionamentos sao:

- `links.empresa_id` referencia `empresas.id`.
- `chamados.empresa_id` referencia `empresas.id`.

## Tabelas e colunas

### `empresas`

- `id` integer, chave primaria, auto increment.
- `nome` varchar(255), nao nulo, unico.

### `links`

- `id` integer, chave primaria, auto increment.
- `empresa_id` integer, nao nulo, chave estrangeira para `empresas.id`.
- `titulo` varchar(255), nao nulo.
- `url` varchar(500), nao nulo.
- `descricao` text, opcional.
- `atualizado_em` timestamp, opcional, padrao `now()`.

### `chamados`

- `id` integer, chave primaria, auto increment.
- `empresa_id` integer, nao nulo, chave estrangeira para `empresas.id`.
- `titulo` varchar(255), nao nulo.
- `descricao` text, nao nulo.
- `status` varchar(50), padrao `Aberto`.
- `observacao_ti` text, opcional.
- `criado_em` timestamp, padrao `now()`.
- `atualizado_em` timestamp, padrao `now()`.

### `agenda_eventos`

- `id` integer, chave primaria, auto increment.
- `titulo` varchar(255), nao nulo.
- `data_inicio` timestamp, nao nulo.
- `data_fim` timestamp, nao nulo.
- `local` varchar(255), opcional.
- `descricao` text, opcional.
- `criado_em` timestamp, padrao `now()`.
- `atualizado_em` timestamp, padrao `now()`.

## SQL pronto para o Supabase

Execute este bloco no SQL Editor do Supabase:

```sql
create table if not exists public.empresas (
  id bigint generated by default as identity primary key,
  nome varchar(255) not null unique
);

create table if not exists public.links (
  id bigint generated by default as identity primary key,
  empresa_id bigint not null references public.empresas (id) on delete cascade,
  titulo varchar(255) not null,
  url varchar(500) not null,
  descricao text null,
  atualizado_em timestamp without time zone default now()
);

create table if not exists public.chamados (
  id bigint generated by default as identity primary key,
  empresa_id bigint not null references public.empresas (id) on delete cascade,
  titulo varchar(255) not null,
  descricao text not null,
  status varchar(50) default 'Aberto',
  observacao_ti text null,
  criado_em timestamp without time zone default now(),
  atualizado_em timestamp without time zone default now()
);

create table if not exists public.agenda_eventos (
  id bigint generated by default as identity primary key,
  titulo varchar(255) not null,
  data_inicio timestamp without time zone not null,
  data_fim timestamp without time zone not null,
  local varchar(255) null,
  descricao text null,
  criado_em timestamp without time zone default now(),
  atualizado_em timestamp without time zone default now()
);

create index if not exists idx_links_empresa_id on public.links (empresa_id);
create index if not exists idx_chamados_empresa_id on public.chamados (empresa_id);
create index if not exists idx_agenda_eventos_data_inicio on public.agenda_eventos (data_inicio);
```

## Configuracao para deploy

### Backend

Defina estas variaveis de ambiente no backend:

- `DATABASE_URL`: connection string do Supabase.
- `DATABASE_SSLMODE=require`: obrigatorio para conexao externa.
- `CORS_ORIGINS`: URL publica do frontend na Vercel.
- `N8N_CHAT_WEBHOOK_URL`: opcional, se quiser manter a integracao do chatbot.

### Frontend na Vercel

Defina:

- `VITE_API_URL`: URL publica do backend.

## Observacoes importantes

- O backend usa `Base.metadata.create_all(...)` em [backend/database.py](backend/database.py), entao o schema acima precisa existir no Supabase antes do primeiro acesso em producao.
- Se quiser manter consistencia total entre app e banco, prefira criar as tabelas exatamente como acima e depois apontar o `DATABASE_URL` para o Supabase.
- O frontend consome apenas os endpoints de API, entao ele nao acessa o Supabase direto.
